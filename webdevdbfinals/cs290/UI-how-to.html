UI Customization Demonstration

I figured this warranted its own demonstration, as it is a major part of customizing your map and needed to be it's own separate how-to. This should be considered a somewhat advanced technique, and you should familiarize yourself with the rest of the API before you do this.

The positioning nodes are as follows, going clockwise:

  • TOP_CENTER
  • TOP_RIGHT
  • RIGHT_TOP
  • RIGHT_CENTER
  • RIGHT_BOTTOM
  • BOTTOM_RIGHT
  • BOTTOM_CENTER
  • BOTTOM_LEFT
  • LEFT_BOTTOM
  • LEFT_CENTER
  • LEFT_TOP
  • TOP_LEFT
Each of these correspond to a place on the map like you see above. This allows you to customize the user interface of your map to your liking. By default, Google Maps has a pretty good UI, but sometimes you may want to remove, add, or customize the location of various components to match a theme you are going for or even to make it more intuitive for yourself or your users. This goes a bit beyond the scope of this demonstration, but I wanted to show what was possible with some fairly basic programming and considerations. Examine the code below and see what you notice. Do you notice the patterns that have been emerging since the beginning of this how-to? Note that almost everything starts with google.maps then is followed by a specific component or parameter suffix, and it continues to chain until you are done.

For example, look at the code in red below. Notice how it has the Control Style, and the Control Position. This tells the Map API to place a horizontal bar at the top center of the map. While you can clutter the API with a lot of controls, it does attempt to place things in a reasonable and intelligble order.

Within the properties, we also have the mapTypeIds, which determine the type of maps that will be available on this map. I have commented out the remaining two; however, these can be used in lieu or in tandem with the available map types. Terrain generally covers the topography of the map, while road maps speak for themselves. A hybrid combines terrain and roadmaps or satellite and roadmaps, and satellite maps give a digital imagery view of the area your map is looking at.


<script type="text/javascript">

function initMap() {
  var map = new google.maps.Map(document.getElementById('map'), {
    zoom: 12,
    center: {lat: 34.01, lng: -118.0},
    mapTypeControl: true,
    mapTypeControlOptions: {
      style: google.maps.MapTypeControlStyle.HORIZONTAL_BAR,
      position: google.maps.ControlPosition.TOP_CENTER,
      mapTypeIds: [
        google.maps.MapTypeId.ROADMAP,
        google.maps.MapTypeId.TERRAIN
        //google.maps.MapTypeId.HYBRID,
        //google.maps.MapTypeId.SATELLITE
      ]
    },
    zoomControl: true,
    zoomControlOptions: { position: google.maps.ControlPosition.RIGHT_CENTER
    },
      scaleControl: true,
      scaleControlOptions: { position: google.maps.ControlPosition.BOTTOM_RIGHT },
      
      streetViewControl: true,
      streetViewControlOptions: {
        position: google.maps.ControlPosition.BOTTOM_CENTER },
      rotateControl: true,
      rotateControlOptions: {
        position: google.maps.ControlPosition.LEFT_CENTER
      }
   });

  var marker = new google.maps.Marker({
    map: map,
    position: {lat: 34.01, lng: -118.5},
    title: "I can see my house!"
  });

}
</script>
