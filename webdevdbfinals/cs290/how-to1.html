Creating Your First Map

Once you are set up and have your API key, you can start creating and setting up maps. A fairly standard map is given in the code sample given below. This code sample is the same that can be found on Google's documentation, but I have taken the liberty of starring things that you will not need to create a map, and highlighting the code that is essential for the basics of a map.

I have taken the liberty of including the basic skeleton of an HTML page as well, so that you can get an idea of what an .hmtl file looks like OR refresh yourself if you are unsure of anything.

<!DOCTYPE html>* // You do not need the lines that I have noted
 with a red * if you already have an HTML page set up.
<html>*
  <head>*
    <style type="text/css"> // Put this in your CSS file or in the head
 of your html/php/js file.
      html, body { height: 100%; margin: 0; padding: 0; }
      #map { height: 100%; }
    </style>*
  </head>*
  <body>*
    <div id="map"></div>
    <script type="text/javascript">

var map;
function initMap() {
  map = new google.maps.Map(document.getElementById('map'), {
    center: {lat: 44.567, lng: -123.283}, // This...
    zoom: 8 //...and this are required a
                                            // map at all!
  });
}

    </script>
    <script async defer
      src="https://maps.googleapis.com/maps/api/js?key=
YOUR_API_KEY&callback=initMap">
    </script>
  </body>*
</html>*

While I have noted things you don't need with red stars and pointed out essential properties in bold red letters, I also want to point out the YOUR_API_KEY. This is where you put your assigned key (that long string of letters), which allows you to access and pull up your maps. While the key itself is NOT required, per se, you cannot do some of the deeper development and customization discussed in the advanced Google documentation without it.

So, what's going on here?

My script has created a map focusing on the center (or near center) of the OSU campus. Go Beavs! In my earlier code sample, I pointed out two things that are essential to get a properly functioning map up and running. These are the zoom and center properties.
The center property serves to anchor the map and point where the center of the image will be. I opted for the OSU Campus in this example, but you can put any latitude or longitude you want and have your map center on your favorite place.

A Potential Pitfall!

Remember to use negatives when inputing information for places south of the equator for latitude and negatives for longitudes west of the Prime Meridian in your center property.Otherwise you might end up in China, Australia, or Europe when you wanted to put your map's center point on the East Coast of the US, Los Angeles, or even South America!

So, for example, if you want to look at Central Oregon/the Corvallis area, you want to put in 44.567N, 123.283W, or lat: 44.56 and lng: -123.283. Remember as well that degrees are divided by minutes and there are 60 degrees in a minute. So, if your coordinates are 34°01'N, 118°28'W (Santa Monica, CA), you would put in 1/60 or .0167 and 28/60, which is roughly .467, 34.017 and -118.467.

The zoom property determines how close in or how far out your map will start. The larger the number, the closer in you will start. If you chose 1 or 2, you will have largely a "global" view, though your map will always be centered on a particular coordinate. However, please note that you can end up with some locational dilution if you zoom in from far away as the map reorients itself with each zoom level. It usually won't be more than a few miles, but if you start your map in say, Rhode Island, and you zoom out real far, you could easily find yourself in Connecticut or Massachusetts if you zoom in rapidly.

If want to start with a zoom level of 1-4, you might want a marker. For most zoom properties, I'd recommend at least 8, but no more than 16. I used 15 in my map above, which gives a fairly close in view. 12 will give a large neighborhood/district/small city view, and 8 gives a regional view.
