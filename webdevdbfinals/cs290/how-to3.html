Stylizing Your Map

Almost everybody has seen the various doodles and gag-maps that have been put out over the past few years, one of the most famous being the Dragon Quest-style map as the 2012 April Fool's Joke. This was done through the customizability of their API, use of overlays and stylistic design, and you can also make stylistic changes to your map in addition to placing markers, giving, and restricting access to the various features that come with the map. You can really do some fun and neat things with it! While the Dragon Quest example is an extreme one with a lot of advanced work put into it, you can still do plenty with the given styling tools.

Here is an example of what can be done using JSON. Remember, you have to define and then use the style property to see any changes when you create your map. Add code like this after your initMap() function. 

var styleArray = [
  {
    "featureType": "road.arterial",
    "elementType": "geometry.fill",
    "stylers": [
      { "color": "#2cbc2b" },
      { "visibility": "on" },
      { "invert_lightness": true },
      { "weight": 3 }
    ]
  },{
    "featureType": "road.highway",
    "elementType": "geometry.fill",
    "stylers": [
      { "visibility": "simplified" },
      { "color": "#a04043" },
      { "weight": 3.4 }
    ]
  }
];

This is, of course, a very limited example demonstration of what the map API stylizing protocols can be used for. With the various stylistic parameters, you can easily customize your map to serve whatever purpose you want -- point out famous landmarks, make local parks stand out with a bright color, or highlight major traffic, water, and other pathways. Here, I upped the saturation to make the colors stand out and highlighted the arterial (major land-based) roadways in the map in green. It is a simplistic example, but it shows some of the basic abilities that map stylization has. Further on in the how-to, I create a sepia colored map to demonstrate what can be done with creative use of the varied parameters.

A List of Parameters For Styling Your Map

New Commands

  • style: You attach this to your map with the name of your array of stylistic choices.

Style Properties - there are literally dozens, but I will highlight the major and most useful ones. For a full list, go to Googles Reference here.

  • featureType: The feature you want to highlight.
    • administrative: shows administrative areas. Sub categories can be used to highlight countries, land parcels, and other properties and cities.
    • all: Highlight all the thing.
    • landscape: Various landscaping items, whether natural or manmade.
    • poi: Highlights points of interest. This includes businesses, government, religious, and medical locations.
      • poi.attraction: Tourist attractions.
      • poi.business: Businesses
      • poi.government: Government buildings
      • poi.medical: Hospitals and Emergency services
      • poi.park: Parks
      • poi.place_of_worship: Churches, temples, mosques.
      • poi.school: Schools and universities.
      • poi.sports_complex: Stadiums, arenas, and other sports complexes.
    • road: Apply the styles to all roads.
        • road.arterial: Major thoroughfares and some ground level highwaysroad.highway: Freeways and highways, such as the interstate and state highways. road.local: Neighborhood roads. Generally not useful, and quite easily clutters up the map.
    • transit: Transit lines, airports, and more.
    • water: Water ways.

      • stylers:

      • all: Apply your stylization to everything.
      • geometry: This is the element of the feature. You can use this to make one particular element a solid color or you can make it two colors to stand out.
        • geometry.fill: Selects only the fill (inside) color.
        • geometry.stroke: Selects only the line color.
      • labels: Selects only the text of a particular feature's label
        • labels.icon: Selects only the icon of the label, i.e; freeway symbols
        • labels.text: Selects only the text of the label
        • labels.text.fill: Fills in the letters of a label.
        • labels.text.stroke: Colors in the lines of letters of a label.

    The Stylers

    • hue: Basic Color. RGB style, i.e: #ffffff for white. This is used to tint your map a certain color.
    • lightness: Has a value between -100 and 100. This indicates how bright the color will be, with 100 being pure white, and -100 being pure black.
    • saturation: Has a value between -100 and 100. Determines the intensity of the color. 100 will create incredibly bright and vibrant colors, while -100 will display the map in shades of grey.
    • gamma: Has a Value between 0.01 and 10.0. 0.0 is neutral and no correction. Gamma is a very abstact way of correcting the brightness of an object. It is non-linear (thus there might not be a whole lot of difference between )
    • invert_lightness: - true or false. This switches your color to a darker or lighter counterpart. Useful if you need to quickly make change but don't want to fiddle with the color scheme too much.
    • visibility: (on, off, simplified) Does it appear on the map? You can turn off roads, highways, and other features if you feel they make your map too noisy or cluttered. Useful especially if you just want to show a static, satellite image, without roads and other features that are overlaid onto the image by default.
    • color: RBG style - similar to hue, but this is the color of the feature. Hue is more often used to modify or change the color scheme. color will color your ENTIRE map if you use it under all elements.
    • weight: a value greater that or equal to zero. This indicates how many pixels the color (especially useful with stroke) line will take up. Be aware, if you have high weight elements overlapping each other, you might get what is called clipping -- which is a sudden drop off of the color in a jarring manner.

    Now that you got the basics of how to stylize your maps. With all of the aforementioned properties and parameters, you can create pretty sophisticated and beautiful maps with enough time and patience. Experimentation is the best way to learn.

    I have included a sepia-style map of the Los Angeles area on this page to demonstrate what can be done with a large number of properties and settings. Just be aware, that creating this many properties can make your map hard to manage and may result in overlaps that are displeasing to look at.
